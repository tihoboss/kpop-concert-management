## 🐛 2. Найденные дефекты (Баг-репорты)

### BUG-001: Пароль передается в открытом виде

| Поле | Значение |
|------|----------|
| **ID** | BUG-001 |
| **Серьезность** | HIGH |
| **Приоритет** | HIGH |
| **Компонент** | Авторизация |
| **Статус** | Открыт |
| **Окружение** | Chrome 120, Windows 11 |

**Описание:** При отправке формы логина пароль передается в открытом виде (plain text) без шифрования

**Шаги воспроизведения:**
1. Открыть DevTools (F12)
2. Перейти на вкладку Network
3. Ввести логин и пароль
4. Нажать "Войти"
5. Посмотреть payload запроса

**Ожидаемый результат:** Пароль должен быть зашифрован или скрыт

**Фактический результат:** Пароль виден в открытом виде

**Рекомендация по исправлению:**
- Использовать HTTPS
- Хэшировать пароль на клиенте перед отправкой
- Использовать bcrypt для хэширования

### BUG-002: Нет валидации дат при создании мероприятия

| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Серьезность** | MEDIUM |
| **Приоритет** | MEDIUM |
| **Компонент** | Мероприятия |
| **Статус** | Открыт |

**Описание:** Можно установить дату окончания раньше даты начала мероприятия

**Шаги воспроизведения:**
1. Создать мероприятие
2. Установить start_date = 2025-12-15 19:00
3. Установить end_date = 2025-12-10 18:00
4. Нажать "Сохранить"

**Ожидаемый результат:** Сообщение об ошибке "Дата окончания должна быть позже даты начала"

**Фактический результат:** Мероприятие успешно создается

**Рекомендация по исправлению:**
- Добавить валидацию на клиенте (JavaScript)
- Добавить валидацию на сервере (Node.js)
- Проверять `end_date > start_date`

### BUG-003: Координатор видит все мероприятия

| Поле | Значение |
|------|----------|

| **Приоритет** | LOW |
| **Компонент** | Права доступа |
| **Статус** | Открыт |

**Описание:** Координатор видит мероприятия всех групп, а не только те, где он назначен

**Ожидаемый результат:** Только мероприятия, где координатор назначен

**Фактический результат:** Все мероприятия системы

**Рекомендация по исправлению:**

- Проверять `coordinator_id` в таблице `event_coordinators`

### BUG-005: Отсутствует валидация email при регистрации

| Поле | Значение |
|------|----------|
| **ID** | BUG-005 |
| **Серьезность** | MEDIUM |
| **Приоритет** | MEDIUM |
| **Компонент** | Регистрация |
| **Статус** | Открыт |

**Описание:** Можно зарегистрироваться с некорректным email (без @, без домена)

**Шаги воспроизведения:**
1. Перейти на вкладку регистрации
2. Ввести email "invalid-email"
3. Заполнить остальные поля
4. Нажать "Зарегистрировать"

**Ожидаемый результат:** Сообщение об ошибке "Введите корректный email"

**Фактический результат:** Регистрация проходит успешно

**Рекомендация по исправлению:**
- Добавить валидацию email на клиенте
- Использовать regex: `/^[^\s@]+@([^\s@]+\.)+[^\s@]+$/`
